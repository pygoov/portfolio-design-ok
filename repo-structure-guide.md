# Структура репозитория

Ниже самая полезная ментальная модель этого репозитория: тут есть `3` рабочих слоя и `1` слой результата.

`site-src/` — это сам сайт: страницы, шаблоны, стили.  
`content/` — это данные о работах и структуре портфолио.  
`media/` — это картинки и файлы для работ.  
`docs/` — это уже собранный сайт, его вручную обычно не трогаем.

## Что где менять

- Если нужно править главную страницу, смотри `site-src/index.njk` и `site-src/site.css`.
- Если нужно править страницу списка работ или разделов, смотри `site-src/work/index.njk`, `site-src/work/sections.njk`, `site-src/work/subsections.njk`.
- Если нужно править шаблон отдельного кейса, смотри `site-src/cases/case-pages.njk`.
- Если нужно добавить или изменить работу, обычно меняется JSON в `content/entries`.
- Если нужно добавить или изменить разделы сайта, меняются `content/taxonomy/sections.json` и `content/taxonomy/subsections.json`.
- Если нужно заменить или добавить изображения, рабочие файлы лежат в `media/source`, а web-версии для сайта в `media/web`.

## Как это связано

- Eleventy берет шаблоны из `site-src`.
- Данные подгружаются через `site-src/_data/portfolio.js`.
- Этот файл читает `content/taxonomy` и `content/entries`, а потом подставляет данные в шаблоны.
- Картинки из `media/web` при сборке копируются в `docs/assets/media` по настройке в `.eleventy.js`.

## Очень практично: что делать в типовых случаях

- `Хочу поправить текст/блок на главной` -> `site-src/index.njk`
- `Хочу поменять общий стиль` -> `site-src/site.css`
- `Хочу добавить новую работу` -> новый JSON в `content/entries` + картинки в `media/web`
- `Хочу добавить новый раздел` -> `content/taxonomy/sections.json` и, скорее всего, `content/taxonomy/subsections.json`
- `Хочу понять, почему работа не появилась` -> проверить, что entry подключен в `subsections.json`, потому что именно там задаются `entryIds`

## Главное правило

Не редактируй `docs` вручную. Это build-output. Меняем исходники и потом запускаем `npm run build`.

## С чего удобно начать

1. Смотреть на `repo-map.md` как на краткую карту.
2. Открыть один реальный пример entry в `content/entries/labels-misofi.json`.
3. Потом открыть `site-src/_data/portfolio.js`, чтобы увидеть, как JSON превращается в страницы.
4. После любых изменений в сайте запускать `npm run build`.
